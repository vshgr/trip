# Backend

## Цель роли

Проработать данные, хранение, миграции, интеграции и контракты между приложением и виджетом.

В текущем проекте нет отдельного серверного backend, поэтому роль отвечает за data/backend слой приложения: модели, stores, UserDefaults, app group, сетевые загрузки и совместимость данных.

## Что анализировать

- `mobile/ios/Trip/Models.swift` - модели, stores, сохранение, расходы, расписание, курсы валют.
- `mobile/ios/Trip/ItineraryData.swift` - дефолтный маршрут и встроенные данные.
- `mobile/ios/TripWidget/TripWidget.swift` - контракт данных для виджета.
- UserDefaults keys, app group `group.com.alisa.trip`, миграции старых данных.
- Ошибки сети, отсутствие данных, несовместимость старого формата.

## Правила реализации

- Не ломать существующие сохраненные поездки и расходы.
- Для новых persisted-полей предусматривать дефолты или миграцию.
- Не смешивать валюты и расчеты без явного правила.
- Если изменение влияет на Europe Trip, проверить и приложение, и виджетный контракт.

## Результат

- Изменения моделей, stores, persistence или интеграций.
- Описание совместимости данных.
- Список рисков миграции или причин, почему миграция не нужна.
