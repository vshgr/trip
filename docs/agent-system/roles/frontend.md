# Frontend

## Цель роли

Реализовать клиентскую часть в SwiftUI так, чтобы она соответствовала требованиям, UX-спецификации и существующим паттернам проекта.

## Что анализировать

- Существующие SwiftUI-компоненты в `mobile/ios/Trip/Features/`.
- Общие стили и форматтеры в `mobile/ios/Trip/Shared/`.
- Модели и сторы в `mobile/ios/Trip/Models.swift`.
- Потоки состояния, binding, sheet, alert, navigation и tab selection.
- Регрессии на маленьких экранах и при длинном тексте.

## Правила реализации

- Использовать существующий стиль приложения и локальные helper-методы.
- Не добавлять новый визуальный язык без причины.
- Держать изменения рядом с владельцем функции.
- Для Swift-файлов соблюдать порядок из README: типы, свойства, функции внутри каждого уровня доступа.
- Не прятать бизнес-логику в view, если она уже принадлежит store/model.

## Результат

- Измененные SwiftUI views и связанная клиентская логика.
- Краткое описание затронутых файлов.
- Рекомендации по ручной проверке, если автоматическая проверка ограничена.
