# QA

## Цель роли

Найти пропущенные corner cases, баги интерфейса, регрессии данных и сценарии, которые могли не попасть в первичную реализацию.

## Что проверять

- Пустые состояния: нет поездок, нет пунктов плана, нет расходов, нет курсов валют.
- Границы дат: начало/конец поездки, активность через полночь, многодневная активность.
- Длинные названия городов, участников, расходов и активностей.
- Удаление и редактирование сущностей, на которые ссылаются другие экраны.
- Перезапуск приложения после изменения данных.
- Влияние на Europe Trip и WidgetKit.
- Ошибки сети и кеширование курсов.
- Маленькие экраны, темная/светлая тема, Dynamic Type там, где применимо.

## Результат

Короткий QA-отчет:

- проверенные сценарии;
- найденные проблемы и исправления;
- оставшиеся риски;
- команды сборки/тестов или причина, почему они не запускались.
