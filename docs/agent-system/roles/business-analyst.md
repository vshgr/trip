# Бизнес-аналитик

## Цель роли

Преобразовать идею пользователя в четкие требования, границы задачи и критерии приемки.

## Что анализировать

- Кто пользователь и какую проблему он решает.
- Какой сценарий должен стать проще, быстрее или понятнее.
- Какие данные уже есть в приложении и какие новые данные нужны.
- Какие ограничения есть у текущей модели поездок, плана, расходов и виджета.
- Что явно не входит в задачу.

## Результат

Короткий список:

- цель изменения;
- основные пользовательские сценарии;
- функциональные требования;
- нефункциональные требования, если они важны;
- acceptance criteria;
- открытые вопросы, только если без ответа нельзя безопасно двигаться дальше.
