# ADR 0003: Хранение денег

## Статус

Принято.

## Решение

Деньги хранятся как `amount_minor BIGINT` и `currency_code CHAR(3)`.

## Обоснование

В iOS сейчас используется `Double`, но backend не должен терять копейки/центы при split-расчетах и балансах.

## Последствия

- Нельзя использовать `float32`/`float64` для денег.
- Равное деление расходов распределяет остаток детерминированно.
- Балансы считаются отдельно по валютам.
- Поддерживаемые валюты MVP: `RUB`, `EUR`, `USD`, `KZT`, `JPY`.
