# Роли и права

## Два уровня ролей

В проекте нужны два независимых уровня прав.

Системная роль пользователя:

- `admin`;
- `user`.

Роль пользователя внутри поездки:

- `owner`;
- `editor`;
- `viewer`.

## Почему так лучше

`admin/user` отвечает за доступ к системе целиком. Например, админ может в будущем смотреть служебную панель, помогать с поддержкой, восстанавливать данные или модерировать.

`owner/editor/viewer` отвечает только за конкретную поездку. Один и тот же человек может быть владельцем одной поездки и зрителем другой.

Если смешать эти уровни, права быстро станут запутанными: системный админ не обязательно владелец каждой поездки, а владелец поездки не должен автоматически становиться админом всей системы.

## Матрица прав

| Действие | admin | user | owner | editor | viewer |
|---|---:|---:|---:|---:|---:|
| Войти в приложение | да | да | - | - | - |
| Читать свои поездки | да | да | да | да | да |
| Создать поездку | да | да | - | - | - |
| Редактировать поездку | через поддержку | только свои | да | частично | нет |
| Мягко удалить поездку | через поддержку | только свои | да | нет | нет |
| Восстановить поездку | да | будущая функция | да | нет | нет |
| Редактировать маршрут | через поддержку | при наличии роли | да | да | нет |
| Редактировать расходы | через поддержку | при наличии роли | да | да | нет |
| Смотреть балансы | при наличии доступа | при наличии доступа | да | да | да |

## Что уже добавлено

- В таблице `users` есть `role` со значениями `admin` или `user`.
- Demo user получает роль `admin`.
- Новые пользователи получают роль `user`.
- `/api/v1/me` и auth responses возвращают роль пользователя.
